## 1. 项目文件结构

### 1.1 总览

一个安卓项目的目录结构

```
你的项目/
├── build.gradle                     ← 根配置（相当于根 package.json）
├── settings.gradle                  ← 仓库配置（从哪里下载依赖包）
├── gradlew / gradlew.bat           ← 构建命令脚本
├── gradle/                          ← 构建工具本体
├── local.properties                 ← 本地 SDK 路径
│
└── app/                             ← 主要代码都在这里！
    ├── build.gradle                 ← 模块配置（相当于模块 package.json）
    └── src/main/
        ├── AndroidManifest.xml      ← 应用声明（权限+入口）
        ├── java/.../                ← Java 源代码
        └── res/                     ← 资源文件（布局+样式+图片）
```

### 1.2 对比前端项目

| 前端项目 | 安卓项目 |
|:---|:---|
| `package.json` | `app/build.gradle`（声明依赖） |
| `src/` 目录 | `app/src/main/` 目录 |
| `.html` 文件 | `res/layout/` 下的 `.xml` 文件 |
| `.css` 文件 | `res/values/` 下的 `.xml` 文件 |
| `.js` / `.ts` 文件 | `java/` 下的 `.java` 文件 |

### 1.3 你的项目文件详解

```
V0/app/src/main/
│
├── AndroidManifest.xml              ← 应用的"身份证"
│   ├── 声明用到的权限（网络、定位、录音）
│   ├── 告诉系统有哪些页面（Activity）
│   └── 配置第三方 API Key（高德地图）
│
├── java/com/example/voicenavigation/   ← Java 代码
│   ├── MainActivity.java              ← 主页面（相当于 index.js）
│   ├── navigation/
│   │   └── NavigationManager.java     ← 导航功能模块
│   ├── stt/
│   │   ├── BaiduSpeechManager.java    ← 语音识别（STT）
│   │   └── BaiduTtsManager.java       ← 语音合成（TTS）
│   └── data/
│       ├── AppDatabase.java           ← 数据库初始化
│       ├── VoiceRecord.java           ← 数据模型（类似 interface）
│       ├── VoiceRecordDao.java        ← 数据库操作方法
│       └── VoiceRecordAdapter.java    ← 列表渲染器
│
├── res/layout/                        ← HTML！页面长什么样
│   ├── activity_main.xml              ← 主页布局（地图+按钮）
│   ├── page_history.xml               ← 历史记录页
│   └── page_settings.xml              ← 设置页
│
├── res/values/                        ← CSS！样式和常量
│   ├── colors.xml                     ← 颜色定义
│   ├── strings.xml                    ← 文字 + API Key
│   └── themes.xml                     ← 主题
│
├── res/menu/                          ← 底部导航菜单
│   └── menu_main_bottom.xml
│
├── res/color/                         ← 导航栏颜色
│   └── bottom_nav_color.xml
│
└── assets/                            ← 静态资源
    └── asr_param.json                 ← 语音识别配置
```

## 2. XML 布局 = HTML

### 2.1 基本写法对比

**HTML：**
```html
<div style="display: flex; flex-direction: column; padding: 16px;">
    <input id="searchBox" placeholder="请输入目的地" />
    <button id="searchBtn">搜索</button>
</div>
```

**XML：**
```xml
<LinearLayout
    android:orientation="vertical"
    android:padding="16dp"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <EditText
        android:id="@+id/et_destination"
        android:hint="请输入目的地"
        android:layout_width="match_parent"
        android:layout_height="wrap_content" />

    <Button
        android:id="@+id/btn_search"
        android:text="搜索"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content" />
</LinearLayout>
```

### 2.2 常见标签对照

| HTML | XML（安卓） | 用途 |
|:---|:---|:---|
| `<div>` | `LinearLayout` | 容器，用来排列子元素 |
| `<input>` | `EditText` | 文本输入框 |
| `<button>` | `Button` | 按钮 |
| `<p>` / `<span>` | `TextView` | 显示文字 |
| `<img>` | `ImageView` | 显示图片 |
| `<ul>` / `<li>` | `RecyclerView` | 列表 |
| `<iframe>` | `MapView` | 嵌入地图（高德） |

### 2.3 布局容器（相当于 display: flex）

安卓最常用的两个布局：

```xml
<!-- 相当于 display: flex; flex-direction: column; -->
<LinearLayout android:orientation="vertical"> ... </LinearLayout>

<!-- 相当于 display: flex; flex-direction: row; -->
<LinearLayout android:orientation="horizontal"> ... </LinearLayout>

<!-- 相当于 display: flex; flex: 1; align-items: center; justify-content: center; -->
<FrameLayout> ... </FrameLayout>
```

### 2.4 常用属性对照

| 安卓 XML 属性 | 前端 CSS | 说明 |
|:---|:---|:---|
| `android:layout_width="match_parent"` | `width: 100%` | 宽度填满父容器 |
| `android:layout_width="wrap_content"` | `width: fit-content` | 宽度随内容 |
| `android:layout_height="match_parent"` | `height: 100%` | 高度填满父容器 |
| `android:padding="16dp"` | `padding: 16px` | 内边距 |
| `android:margin="8dp"` | `margin: 8px` | 外边距 |
| `android:gravity="center"` | `text-align: center` + `align-items: center` | 内容居中 |
| `android:layout_weight="1"` | `flex: 1` | 按比例分配空间 |
| `android:visibility="gone"` | `display: none` | 隐藏 |
| `android:visibility="visible"` | `display: block` | 显示 |
| `android:background="#FF0000"` | `background: red` | 背景色 |
| `android:textSize="16sp"` | `font-size: 16px` | 字号 |
| `android:textColor="@color/red"` | `color: red` | 文字颜色 |

### 2.5 `@` 符号的含义

```xml
@+id/btn_search    ← 创建新 id（类似 HTML 的 id="btn_search"）
@id/btn_search     ← 引用已有的 id
@color/red         ← 引用颜色（来自 colors.xml）
@string/app_name   ← 引用字符串（来自 strings.xml）
@color/gray        ← 引用 colors.xml 里定义的 gray 颜色
```

**`@+id/xxx` 就相当于前端写的 `id="xxx"`**，Java 代码里用 `findViewById(R.id.xxx)` 来获取这个元素。

## 3. Java 代码 = JavaScript

### 3.1 获取页面元素

```java
// Java
Button btnSearch = findViewById(R.id.btn_search);
EditText etInput = findViewById(R.id.et_destination);
TextView tvStatus = findViewById(R.id.tv_status);
```

```javascript
// JavaScript
const btnSearch = document.querySelector('#btn_search');
const etInput = document.querySelector('#et_destination');
const tvStatus = document.querySelector('#tv_status');
```

### 3.2 事件绑定

```java
// Java
btnSearch.setOnClickListener(v -> {
    // 这里的 v 就是被点击的按钮
    String text = etInput.getText().toString();
    searchDestination(text);
});
```

```javascript
// JavaScript
btnSearch.addEventListener('click', (event) => {
    const text = etInput.value;
    searchDestination(text);
});
```

### 3.3 读写内容

```java
// Java - 读取输入框内容
String text = etInput.getText().toString();

// Java - 设置文字
tvStatus.setText("已找到 10 个结果");
etInput.setText("北京西站");

// Java - 设置光标位置
etInput.setSelection(text.length());  // 光标放到末尾
```

```javascript
// JavaScript - 读取
const text = etInput.value;

// JavaScript - 设置
tvStatus.innerText = "已找到 10 个结果";
etInput.value = "北京西站";
```

### 3.4 显示/隐藏元素

```java
// Java
view.setVisibility(View.VISIBLE);   // 显示（display: block）
view.setVisibility(View.GONE);      // 隐藏（display: none）
view.setVisibility(View.INVISIBLE); // 隐藏但占位（visibility: hidden）
```

```javascript
// JavaScript
view.style.display = 'block';   // 显示
view.style.display = 'none';    // 隐藏
view.style.visibility = 'hidden'; // 隐藏但占位
```

### 3.5 弹窗

```java
// Java - 相当于 alert()
Toast.makeText(this, "请输入目的地", Toast.LENGTH_SHORT).show();

// Java - 选择列表弹窗
new AlertDialog.Builder(this)
    .setTitle("选择目的地")
    .setItems(["选项1", "选项2"], (dialog, which) -> {
        // 点击第 which 项
    })
    .show();
```

```javascript
// JavaScript
alert("请输入目的地");

// 选择列表
const result = confirm("确认选择？");
```

### 3.6 线程（Java 比较特别的地方）

安卓不能在主线程做耗时操作（比如网络请求、数据库读写），要开新线程：

```java
// Java - 在后台线程做耗时操作
new Thread(() -> {
    // 数据库查询（慢操作）
    List<VoiceRecord> records = database.dao().getAll();

    // 回到主线程更新 UI
    runOnUiThread(() -> {
        textView.setText("查询到 " + records.size() + " 条记录");
    });
}).start();
```

```javascript
// JavaScript - await 一步到位
const records = await database.getAll();
textView.innerText = `查询到 ${records.length} 条记录`;
```

> 注意：JavaScript 是单线程 + 事件循环，没有这个问题。Java 安卓需要手动管理线程。

### 3.7 日志打印（console.log）

```java
// Java
Log.d("标签", "这条信息的内容");
Log.e("标签", "错误信息");

// 在 Android Studio 的 Logcat 中查看
```

```javascript
// JavaScript
console.log("这条信息的内容");
console.error("错误信息");
```

## 4. Gradle 依赖 = npm

### 4.1 依赖声明

```groovy
// app/build.gradle 中的 dependencies 块
dependencies {
    // 就像 npm install 高德地图 SDK
    implementation 'com.amap.api:3dmap:9.7.0'

    // 就像 npm install 高德搜索 SDK
    implementation 'com.amap.api:search:9.7.0'

    // 就像 npm install room
    implementation 'androidx.room:room-runtime:2.6.1'

    // 就像 npm install -D room-compiler（编译期工具）
    annotationProcessor 'androidx.room:room-compiler:2.6.1'

    // 就像 npm install material-ui
    implementation 'com.google.android.material:material:1.11.0'

    // 本地 AAR 文件（就像 npm 安装本地的 .tgz 包）
    implementation(name: 'bdasr', ext: 'aar')
}
```

### 4.2 依赖仓库（registry）

```groovy
// settings.gradle — 就像配置 npm registry
dependencyResolutionManagement {
    repositories {
        google()                    // Google 官方仓库
        mavenCentral()              // Maven 中央仓库
        maven { url '...' }         // 自定义仓库（类似阿里云镜像）
        flatDir { dirs 'libs' }     // 本地文件夹
    }
}
```

### 4.3 常见命令对比

| 操作 | npm | Gradle |
|:---|:---|:---|
| 安装依赖 | `npm install` | 自动下载（或点"Sync Now"） |
| 构建项目 | `npm run build` | `./gradlew assembleDebug` |
| 运行项目 | `npm start` | 点"Run"按钮 |
| 清理缓存 | `rm -rf node_modules` | `./gradlew clean` |
| 安装本地包 | `npm install ./xxx.tgz` | `implementation(name:'xxx', ext:'aar')` |

### 5.1 前端 vs 安卓 完整对照

### 5.3 条件判断和循环

```java

if (records == null || records.isEmpty()) {

for (int i = 0; i < 10; i++) {

}
```

## 6. 你的项目代码运行流程

### 6.1 App 启动顺序

```
1. 用户点击 App 图标
         ↓
2. AndroidManifest.xml 被读取
   - 检查权限声明
   - 找到 MainActivity 作为入口
         ↓
3. MainActivity.java 的 onCreate() 执行
         ↓
4. setContentView(R.layout.activity_main)
   → 解析 activity_main.xml 布局
   → 创建所有按钮、输入框、地图
         ↓
5. initViews()
   → findViewByid 找到所有 UI 元素
   → 绑定点击事件
         ↓
6. initServices()
   → 初始化语音识别、导航、数据库
         ↓
7. 地图加载完成 → 定位到用户当前位置
   → 等待用户操作
```

## 7. 常用工具速查

### 7.1 如何看错误日志（console.log）

1. 手机连电脑，运行 App
2. 在 Android Studio 底部点击 **Logcat** 选项卡
3. 搜索栏输入标签名，例如 `MainActivity`、`NavigationManager`

### 7.2 如何重新安装 App

当改了很多代码，或者数据库结构变了：
1. 在手机上长按 App 图标 → 应用信息 → 清除数据
2. 或在 Android Studio 重新点 Run

### 7.3 常用快捷键

| 操作 | 快捷键 |
|:---|:---|
| 运行 App | `Shift + F10` |
| 查找类/文件 | `Ctrl + N` |
| 搜索全部 | 双击 `Shift` |
| 格式化代码 | `Ctrl + Alt + L` |
| 自动导包 | `Alt + Enter` |
| 显示 Logcat | 点击底部 Logcat 标签 |

### 7.4 常见错误及含义

| 报错信息 | 含义 | 通常原因 |
|:---|:---|:---|
| `NullPointerException` | 空指针异常 | 某个对象是 null 但调用了它的方法 |
| `Resources.NotFoundException` | 资源找不到 | XML 里引用了不存在的 id |
| `ActivityNotFoundException` | Activity 找不到 | AndroidManifest 没注册 |
| `SecurityException` | 权限拒绝 | 没在 Manifest 声明或用户没授予 |
| `ClassCastException` | 类型转换错误 | 把 A 类型当 B 类型用 |
| `No layout manager attached` | 列表缺少布局管理器 | RecyclerView 没 setLayoutManager |

## 写在最后

### 安卓开发的核心思维转变

| 前端习惯 | 安卓需要知道 |
|:---|:---|
| 页面路由（Vue Router / React Router） | Activity 跳转或 Fragment 切换 |
| CSS 直接写样式 | 用 XML 属性或 style 文件 |
| `npm install 库` | `build.gradle` 加一行依赖 |
| `async/await` 做异步 | 回调、线程、Handler |
| 浏览器兼容性 | 手机品牌/系统版本兼容性 |
| Chrome DevTools | Logcat |

**最重要的三点：**

1. **XML 写界面，Java 写逻辑** — 就像 HTML + JavaScript
2. **`findViewById` 是一切交互的起点** — 相当于 JS 的 `querySelector`
3. **Logcat 是你的好朋友** — 遇到问题先看日志
